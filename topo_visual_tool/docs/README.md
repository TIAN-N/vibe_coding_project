# 拓扑可视工具

一个轻量级 Web 前端拓扑可视页面，用于在本地 PC 直接打开并完成网元、链路数据的上传、解析、GIS 展示、逻辑拓扑展示、定位、高亮、过滤和在线数据管理。

## 快速启动

直接双击打开：

```text
topo_visual_tool.html
```

页面没有构建步骤，不需要安装 npm 依赖。也可以在当前目录启动任意静态文件服务后访问 `topo_visual_tool.html`。

## 数据格式

### 网元表

支持 CSV、XLSX、XLS。必选字段：

| 字段 | 说明 |
| --- | --- |
| `NE Name` | 网元名称，作为唯一匹配键 |
| `Role` | 网元角色，支持 `CSG`、`ASG`、`PE` |
| `Longitude` | 经度 |
| `Latitude` | 纬度 |

其他字段会被自动识别，可用于高亮、过滤和数据管理，例如 `Region`、`Vendor`、`Status`、`Site Type`、`Service Level`。

### 链路表

支持 CSV、XLSX、XLS。必选字段：

| 字段 | 说明 |
| --- | --- |
| `Src NE Name` | 源网元名称 |
| `Sink NE Name` | 宿网元名称 |

每行代表一条双向逻辑连接。链路端点需要能匹配网元表中的 `NE Name`。

可选字段：

| 字段 | 说明 |
| --- | --- |
| `Route WKT` | GIS 路网路径，支持 `LINESTRING(lng lat, lng lat, ...)`；为空或不存在时不绘制路网路径 |

## 已实现能力

- 上传网元表和链路表，前端自动解析 CSV/XLSX。
- 内置大规模 Mock 数据：约 120 个网元和 177 条链路，用于功能验收和中等规模性能观察。
- 已导出 Mock CSV 文件：`device.csv` 和 `link.csv`，可从空页面开始手动上传测试。
- GIS Topo：按经纬度在地图上绘制网元与链路。
- GIS 路网路径：link 表存在非空 `Route WKT` 时，按路由折线绘制路网路径，并支持显示开关、颜色、线宽和线型配置。
- GIS 路网路径交互：点击路网路径可高亮对应路由、源宿直连链路和源宿网元；点击空白区域清除当前选中态。
- GIS 路网路径透明度：路网路径样式支持透明度滑杆，便于在底图、直连链路和真实路由之间调节可见度。
- 复合条件：高亮、过滤、节点样式规则和链路样式规则支持多字段条件组合，并可从历史条件一键恢复。
- 样式规则应用：节点样式规则和链路样式规则均采用编辑后点击应用才生效的交互。
- 高亮聚焦：高亮命中对象保留原节点/链路样式，仅淡化未命中对象，并支持自定义对比度。
- GIS 可视强化：地图底图降噪，链路双层描边，节点使用简洁描边和角色色填充。
- GIS 稳定性优化：缩放/拖动后防抖重绘，在线瓦片连续失败时自动切换为本地浅色简洁底图。
- Logic Topo：在白色 SVG 画布上绘制逻辑连接关系，支持拖拽节点、平移和缩放。
- Logic Topo 采用类似 NetworkX `spring_layout` 的浏览器端弹簧布局，并使用网络结构化坐标作为初始 seed。
- Spring layout 同时考虑链路弹簧力、节点斥力、弱角色层级约束和标签预留空间。
- Logic Topo 会将当前范围内没有连接的网元从 spring layout 中拆出，统一平铺到画布最下层，避免散点影响有连接节点的松散布局。
- 点击任一节点后，会强调该节点的一跳直连链路和邻居节点，便于查看具体连接关系。
- Logic 节点大小按自由度自适应，连接越多节点越大。
- 选中和高亮样式优化：不再使用突兀黑色边框和节点外圈阴影，改为简洁描边强调。
- 样式设置：高亮节点支持设置节点颜色、节点大小、节点形状和相关链接颜色；节点图例默认按角色配置基础颜色、大小、形状。
- 节点形状：支持圆形、方形、菱形、三角形；GIS 与 Logic Topo 同步生效。
- 节点样式规则：独立于角色图例，用户可新增任意 device 字段条件规则；点击“应用节点规则”后覆盖命中节点，支持上移/下移调整优先级，删除规则或重置样式后恢复角色图例基础样式。
- 链路样式规则：自动解析 link 表字段和字段枚举值，支持多条规则按字段条件配置链路颜色、实线/虚线/点线、细/中/粗线宽。
- 字段值联想：链路样式规则、高亮条件值、过滤条件值会根据当前选择字段自动联想上传数据中的已有取值，同时保留手动输入能力。
- 网元定位：输入网元名称，支持联想匹配并定位到目标节点。
- 高亮条件：动态读取网元表字段，按条件高亮网元及相关链路。
- 过滤条件：动态读取网元表字段，保留符合条件的网元，以及源端或宿端任一端命中的链路；链路另一端网元作为上下文显示。
- 搜索历史缓存：网元定位、高亮条件值、过滤条件值会记录最近 12 条本地历史，清空输入后再次点击输入框会浮现历史下拉候选。
- 批量查询：支持从 Word / Excel / CSV 粘贴网元或链路清单，点击“查询”后只显示相关网元、相关链路和上下文端点。
- 大规模数据保护：针对 2w+ 网元和万级链路，上传后建立索引，Logic Topo 按过滤/查询结果限量布局，GIS 按视窗限量渲染并限制单次绘制规模，数据表只渲染前 500 行。
- 数据管理页签：查看网元表/链路表，支持表格过滤、统计、输入框编辑、新增和删除行。
- 中英文 UI 切换：中文 / English，英文中“网元”统一使用 `Device`，“链路”统一使用 `Link`。

## 外部依赖

页面通过 CDN 加载以下浏览器库：

- Leaflet：GIS 地图与地理图层渲染。
- SheetJS：XLSX/XLS 文件解析。
- OpenStreetMap 瓦片：GIS 在线底图；瓦片失败较多时自动退化为本地浅色简洁底图。

如果没有网络：

- CSV 解析、Mock 数据、Logic Topo、数据管理仍可使用。
- XLSX/XLS 解析不可用。
- GIS 在线底图不可用时会自动切换为浅色简洁底图，仍可查看经纬度位置、网元和链路。

当前严格要求字段名与需求一致。若后续需要支持别名，例如 `Name` 映射到 `NE Name`，可在 `normalizeRow` 或新增字段映射函数中处理。

当前数据只保存在浏览器内存中。后续可选方向：

- 使用 `localStorage` 保存最近一次上传和编辑结果。

- 引入专业图布局库。
- 增加折叠汇聚层、按环折叠、按链路类型显隐等布局模式。

## 大规模数据说明

已生成性能测试数据：

- `large_device.csv`：20000 条网元，分布在泰国多个区域。
- `large_link.csv`：17500 条链路。

当前大规模数据构造规则：

- 不构造 PE。
- 2500 个 ASG，17500 个 CSG。
- 每个 ASG 连接附近 7 个 CSG，满足“4-10 个 CSG”的约束。
- 每个 CSG 只有 1 条连接，满足“至多 2 条连接”的约束。
- ASG-CSG 距离控制在约 20km 以内。

大规模数据下，工具不会一次性全量绘制 Logic Topo。推荐流程：

1. 上传 `large_device.csv` 和 `large_link.csv`。
2. 等待页面提示大规模数据已加载。
3. 使用网元定位、过滤条件或批量查询缩小范围。
4. 在范围低于 600 个网元、1000 条链路后查看 Logic Topo。
5. GIS Topo 会按当前地图视窗和渲染上限显示局部对象；缩放或拖动时会延迟到地图稳定后刷新业务图层。

本轮大规模校验结果：

- 解析 `large_device.csv` / `large_link.csv` 并建立索引：20000 网元、17500 链路，断链 0，PowerShell 验证耗时约 3 秒。
- 连接度校验：CSG 最大连接数 1，ASG 最小/最大连接数均为 7，符合当前测试拓扑约束。
- GIS 抽样渲染规模：1500 个有效坐标网元，匹配当前 GIS 单次渲染上限。
- Logic 局部样本：360 网元 / 315 链路低于阈值，可进入布局；另构造 550 网元 / 479 链路局部样本验证孤立节点分流，孤立节点进入底部散点带。

## 2026-07-01 当前版本能力补充

本节汇总最近几轮功能拓展后的当前能力，作为 README 的最新准确信息。

### 项目名称记录

- 顶部右上角、语言切换控件前新增“项目名称”输入栏。
- 每次成功解析上传数据或加载 Mock 数据后，自动写入解析时间戳。
- 默认格式为 `YYYY-MM-DD-HH-mm-ss`，例如 `2026-07-01-23-05-09`。
- 用户可以手动修改项目名称，用于标记当前数据版本；语言切换不会覆盖已输入名称。

### 上传数据

必传：

- 网元表：需要 `NE Name`、`Role`、`Longitude`、`Latitude`。
- 链路表：需要 `Src NE Name`、`Sink NE Name`。

可选：

- 环链识别表：需要 `Category`、`Name`、`Root1`、`Root2`、`Label`、`Member_num`、`Member_path`、`Uplink_pair`、`Belong_agg`。
- 链路表中的 `Route WKT`：存在且非空时，在 GIS 地图上绘制真实路网路径。

环链识别表说明：

- `Category=Ring` 计入环数量。
- `Category=Link` 计入链数量。
- `Member_path` 按 `->` 解析，例如 `A->B->C`。
- 如果 `Member_path` 中有网元不在网元表中，页面给出 warning，但不中断解析；缺失网元在后续高亮、过滤和样式计算中忽略。
- 仓库内提供 `ring_chain_sample.csv`，基于当前 `device.csv` 和 `link.csv` 构造，可用于环链功能测试。
- 当过滤条件来源为环链字段时，统计卡片会额外显示“当前显示环”和“当前显示链”，分别统计本次环链条件命中的 `Category=Ring` 和 `Category=Link` 记录数。

### 高亮与过滤

当前高亮和过滤均支持复合条件：

- 支持多字段条件组合。
- 支持“全部满足”和“任一满足”。
- 条件值输入框会自动合并当前数据枚举值和历史输入值。
- 历史复合条件可在弹窗中一键恢复。

条件来源：

- 网元字段：直接匹配 device 表记录。
- 链路字段：先匹配 link 表记录，再从命中链路的 `Src NE Name` 和 `Sink NE Name` 提取网元集合。
- 环链字段：先匹配环链表记录，再从命中记录的 `Member_path` 提取网元并集。

过滤逻辑：

- 先得到命中网元集合。
- 只展示命中网元。
- 链路必须满足源端和宿端都在命中网元集合内。
- 当条件来源为链路字段时，只保留命中链路表条件的链路及其源宿网元。
- 不再自动带入不符合条件的一跳上下文网元。

高亮逻辑：

- 先得到命中网元集合。
- 源端和宿端都在命中网元集合内的链路视为命中链路。
- 当条件来源为链路字段时，只有命中链路表条件的链路视为命中链路，其源宿网元视为命中网元。
- 命中对象保留当前节点/链路样式。
- 未命中对象按“对比度”配置降低透明度；对比度越高，未命中对象越淡。

### 样式规则

节点样式规则：

- 基于 device 表字段配置。
- 支持颜色、大小、形状。
- 编辑后需要点击“应用节点规则”才生效。

链路样式规则：

- 基于 link 表字段配置。
- 支持颜色、线型、线宽。
- 编辑后需要点击“应用链路规则”才生效。

环链样式规则：

- 仅在上传环链识别表后展示。
- 基于环链表字段配置复合条件。
- 只作用于命中环链记录 `Member_path` 中相邻节点形成的路径段。
- 例如 `A->B->C` 只作用于 `A-B` 和 `B-C`。
- 多条规则命中同一链路时，后应用规则覆盖先应用规则。

### 路网路径图层

当 link 表存在 `Route WKT` 且内容非空时，GIS 视图自动绘制路网路径：

- 支持 `LINESTRING(lng lat, lng lat, ...)`。
- 支持颜色、线宽、线型、透明度配置。
- 默认勾选显示路网路径。
- 取消勾选后不显示路网路径。
- 点击某条路网路径时，会高亮该路由、对应源宿直连链路，以及源宿网元。
- 若路网路径起终点与源宿网元经纬度不完全重合，会绘制引导线辅助识别。
- 地图缩放/拖动时会移除旧 Canvas 图层并在地图稳定后重绘，降低残影和卡顿。
