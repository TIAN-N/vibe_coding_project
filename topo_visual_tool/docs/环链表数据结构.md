- 文件名: ring_chain_division_20260630192221.csv
- 记录数: 11,911 条（不含表头）
- 分隔符: 逗号 (,)
- 引用符: 双引号 (")

## 字段分析

### 1. Category

- 含义: 环链类型分类
- 数据类型: 字符串
- 示例值: `Ring`
- 取值范围: 固定值 `Ring`

### 2. Name

- 含义: 环链的唯一标识名称
- 格式: `{端点1}***{端点2}***{序号}`
- 示例值: `BKFC005-AG-01***BKK059N-AG-01***6484`

### 3. Root1

- 含义: 环链的第一个端点节点
- 格式: `{站点代码}-AG-{序号}`
- 示例值: `BKFC005-AG-01`

### 4. Root2

- 含义: 环链的第二个端点节点
- 格式: 同 Root1
- 示例值: `BKK059N-AG-01`
- 注意: 当 Label 为 `单归环` 时，Root2 与 Root1 相同

### 5. Label

- 含义: 环的类型标签
- 取值范围:
  - `接入环`: 接入层双归环，双端点不同
  - `开环`: 单归属不完全环，双端点不同
  - `单归环`: 环起点和终点为同一节点，Root1 == Root2

### 6. Member_num

- 含义: 环的成员节点数量
- 数据类型: 整数
- 取值范围: `3` ~ `6`
- 说明:
  - 3: 端点 + 1个中转节点
  - 4: 端点 + 2个中转节点
  - 5: 端点 + 3个中转节点
  - 6: 端点 + 4个中转节点

### 7. Member_path

- 含义: 从 Root1 到 Root2 的完整路径
- 格式: `{节点1}->{节点2}->...->{节点N}`
- 分隔符: `->`
- 示例值: `BKFC005-AG-01->BKFC024-AC-01->BKK059N-AG-01`
- 节点类型:
  - `-AG-`: 接入网关节点 (Access Gateway)
  - `-AC-`: 接入交换机节点 (Access Switch)
  - `-AC-_POI`: 带POI标记的接入交换机节点

### 8. Uplink_pair

- 含义: 上行链路节点对
- 格式: `{节点1}***{节点2}`
- 示例值: `BKFC024-AC-01***BKFC024-AC-01`
- 注意:
  - Member_num=3 时，两个节点通常相同
  - Member_num>=4 时，两个节点不同

### 9. Belong_agg

- 含义: 环所属的聚合信息
- 格式: `{节点1}***{节点2}`
- 示例值: `BKFC005-AG-01***BKK059N-AG-01`
- 说明: 通常与 Root1 和 Root2 的值一致

## 字段间关系

### Name 与 Root1/Root2 的关系

```
Name = Root1 + "***" + Root2 + "***" + 序号
```

### Label 与端点的关系

- `接入环`: Root1 ≠ Root2
- `开环`: Root1 ≠ Root2
- `单归环`: Root1 == Root2

### Member_num 与 Member_path 的关系

```
Member_num = Member_path 中节点数量 = Count(节点列表)
```
