# 接入层孤点 CSG 组网优化 —— 算法设计方案

> 目标：为无链接的新增 CSG 添加 IP 连接，使其接入现网并具备可靠性。
> 口径演进：①双归=到 ASG 存在两条**边不相交**路径（可同 ASG）；②**环长≤10 改为软约束**（惩罚，非硬限）；
> ③新增 CSG 度=2；④新增组网有**三种方式按成本择优**；⑤现网既有单归节点也**一并整改**。
>
> ⚠️ 下方"§0 现行算法(v3)"为**当前实现**；§1–§8 为最初 v1 设计思路(成环+硬约束+最大流预言机)，保留作设计沿革。

---

## 0. 现行算法（v3，对应 code/optimize.py）

### 0.1 三种组网方式（均保证新增 CSG 双归，按"新增光缆成本"自动择优）
统一抽象：把"一条新增链(1+个新增CSG)的两端，接到两个锚点"。差别只在锚点来源：

| 方式 | 锚点 | 双归来源 | 约束 |
|---|---|---|---|
| **fiber_extension** | 两台**现网既有双归 CSG**（双锚绕接，不断边） | 经两锚分两方向到 ASG | 链长≤`MAX_FIBER_CHAIN`(3)；每台设备新增邻居≤`MAX_NEW_PER_DEVICE`(3) |
| **loop_insertion** | 某条规整环边（splice 断边，断边不计成本） | 落在环上 | — |
| **new_loop** | 最近一对 ASG（自成双 ASG 环） | 自有环经两 ASG | — |

> 可锚的"现网既有双归 CSG"= 规整环 + **环带链(分支环)的环上 CSG**——后者虽因含分支被"冻结"不可插入，但其环上节点是双归的、可作 fiber 锚点（关键：避免贴近分支环的新增点被误判为远端）。

### 0.2 主流程（聚簇 → 拆链 → 三方式择优 → 整改 → 校验）
1. **盘点**：识别 ASG / 孤点(度0) / 规整环 / 不规整(分支)冻结分量；用 2-边连通求"现网既有双归 CSG"集合。
2. **聚簇**：并查集把互相成本 ≤`D_CLUSTER` 的孤点并为一簇；簇内最近邻+2-opt 成链。
3. **远近判定(规则B)**：簇到最近"现网既有双归 CSG"的成本 ≤`D_FARNESS` 记为**贴近**，否则**远端**。
   - 贴近簇 → 拆成 ≤`MAX_FIBER_CHAIN` 的子链，**就近 fiber_extension / loop_insertion 入网**；
   - 远端簇 → 拆成 ≤`RING_SOFT_MAX-2` 的子链，**new_loop 自成新环**（控制环规模）。
4. **每子链择优**：在可行方式中取 `内部成链成本 + 锚接成本 + 环长惩罚` 最小者。
5. **环长软约束(规则C, 边际)**：惩罚 = `LAMBDA_RING · max(0, 成环后节点数 − max(软上限, 当前环大小))`
   —— 只罚"被本次新增推到软上限/当前规模之上"的部分，避免对本就>10 的现网大环一刀切重罚而倒逼 new_loop。
   用并查集 `RingTracker` 动态跟踪成环规模(loop 增长 / fiber 跨环合并)。
6. **现网既有单归整改**：识别既有单归 spur，给自由端补一条到最近"既有双归设备/ASG"的链路闭合成环(方式 `remediation`)；
   锚点排除新增孤点(保其度=2)；回滚试探须先判 `has_edge` 避免误删原边。
7. **校验**（O(V+E)）：双归判定不再用每点最大流，而用 **2-边连通**——把 `{超汇t}∪ASG` 串成环作 2-边连通核，
   去桥(`nx.bridges`)后不在 t 分量的 CSG 即单归。校验三项：新增孤点双归 / 新增孤点度=2 / 整改后全员双归。

### 0.2.1 输入数据 schema
- `device.csv`：`NE Name, Role(asg/csg), Longitude, Latitude, Is_new_ne`。
  `Is_new_ne=1` 标识**新增孤点 CSG**，`=0` 为现网既有网元（ASG/现有 CSG/分支 CSG，含极少数未挂环的现网 ASG）。
- `link.csv`：`Src NE Name, Sink NE Name`（现网既有链路）。
- `road_distance.csv`：`Src NE Name, Sink NE Name, Distance`（仅 CSG-CSG 且直线≤3km，单位米）。

### 0.3 成本模型
`cost(u,v) = road_distance(u,v)`（命中 road_distance.csv，仅 CSG-CSG≤3km）`否则 = haversine 直线距离`。
仅计**新增光缆**；splice 断开的旧光缆不计。

### 0.4 关键可配置参数（code/optimize.py 顶部）
`MAX_FIBER_CHAIN=3, MAX_NEW_PER_DEVICE=3, D_CLUSTER=1200, D_FARNESS=3000, RING_SOFT_MAX=10, LAMBDA_RING=800`。

### 0.5 复杂度与规模
聚簇 O(n²) 但 n=孤点数；最近锚/farness 用**空间网格**加速；校验/整改用 O(V+E) 桥判定。
test_4（120 ASG + 3000 现网 + 1200 新增, 4407 CSG）端到端约 **27s**, 全部校验通过。

### 0.6 已知权衡
"短纤就近入网"会让大量 fiber 绕接同一现网环、撑大其 2-边连通域（test_4 最大环达 68）。
软惩罚不足以在"短纤(便宜) vs 远端 new_loop(贵)"间扭转。要硬控环规模需**按环容量分流**（见 WORKLOG 待办）。

## 1. 问题定义与数学建模

### 1.1 网络模型与符号

- 图 `G = (V, E)`：
  - `V = A ∪ C`，`A` 为 ASG 集合（汇聚/上联点），`C` 为 CSG 集合（基站侧）。
  - `C = C_old ∪ C_new`：`C_old` 为已在现网环上的 CSG，`C_new` 为待接入的**孤点**（度=0）。
  - `E_0`：现有链路（link.csv），视为**固定且免费**。
- 待求解：新增链路集合 `A_new`。目标是用最小成本把 `C_new` 接入，并满足下述可靠性与环长约束。
- 每条候选链路 `(u,v)` 的成本：
  ```
  cost(u,v) = road_dist(u,v)                    若 (u,v) ∈ road_distance.csv
            = α · haversine(GIS_u, GIS_v)        否则（GIS 直线距离 × 绕行系数 α，建议 α≈1.3）
  ```
  并设置最大可连距离门限 `D_max`：`cost(u,v) > D_max` 的候选边直接剔除（工程上不会拉这么远的纤）。

### 1.2 可靠性约束的精确刻画（口径②）

"每个 CSG 到 ASG 有两条边不相交路径"，等价于：
> 将所有 ASG 合并为一个超级汇点 `t`，每条边容量置 1，则对每个 `v ∈ C` 必有 `maxflow(v → t) ≥ 2`。

这就是构造的**可行性判定预言机（feasibility oracle）**，对任何解都能精确验证（含"插入现有环"后的拼接是否破坏了别人的双归）。
> 注：环（cycle）天然满足该条件——只要环上经过 ≥1 个 ASG，环上每个 CSG 沿两个方向都能到达 ASG，恰为两条边不相交路径。因此**"成环"是满足口径②的自然实现方式**。

### 1.3 环长约束（口径①）

每个环（含其上的 ASG 端点）节点数 ≤ 10。
- 单 ASG 闭合环：`ASG—c1—…—ck—ASG`（ASG 两个端口各出一条边），则 `k ≤ 9` 个 CSG。
- 双 ASG 环：`ASG_A—…—ASG_B—…—ASG_A`，总节点 ≤ 10。
- 向现有环插入 1 个孤点：环节点数 +1，要求插入后仍 ≤ 10。

### 1.4 "插入现有环"的链路语义

物理上把孤点 `x` 插到现有环边 `(a,b)` 之间 = 断开 `a-b`、新增 `a-x`、`x-b`（splice 拼接）。
- 这是把节点接入环的唯一不破坏环结构的方式，本方案按此处理。
- 成本计：`Δcost = cost(a,x) + cost(x,b) − cost(a,b)`（若 `a-b` 为现有免费边，则简化为 `cost(a,x)+cost(x,b)`，按你后续口径定）。

### 1.5 目标函数

```
min  Σ cost(e)   over e ∈ A_new
```
次要目标（字典序/加权）：① 新增链路总长最短 → ② 单环跳数尽量小（负载/时延）→ ③ GIS 直线距离做平局裁决。
约束：环长 ≤ 10、每个 CSG 双归（maxflow≥2）、节点度 ≤ 端口上限（默认 CSG 环上度=2）。

> 该问题是**容量受限的环装载 / m-Ring-Star / SONET 环设计**的变体，属 NP-hard（含 CVRP 为特例）。因此采用"构造启发式 + 局部搜索"为主、小规模可选 MILP 精确求解的方案。

## 2. 总体方案（Pipeline）

```
S0 数据/图构建
   └─ 解析 device/link/road_distance，识别 A / C_old / C_new，定位现有环
S1 候选边生成（稀疏化）
   └─ 每个孤点取 kNN（按 cost），加 D_max 门限，得候选边集
S2 现网盘点
   └─ 标定各现有环的剩余容量(10−当前节点数)、环的 ASG 锚点
S3 构造式启发式（核心）
   └─ 容量受限"最便宜插入"：孤点优先插入最近的、未满的现有环；
      插不进的孤点彼此聚类，新建锚定 ASG 的环
S4 局部搜索改进
   └─ 2-opt / Or-opt / 跨环迁移，降总成本，保持约束
S5 可行性校验（预言机）
   └─ 超汇 max-flow=2 全量校验；不满足处定向修复
S6（可选）MILP 精确求解
   └─ 对单个聚类/小子网做最优环设计，与启发式对照
S7 输出
   └─ 新增 link 行（含成本、所属环、是否 splice）+ 校验报告 + 可视化(可选)
```

## 3. 各阶段算法细节

### S0 数据与图构建
- 读 device.csv → 节点表（NE Name, Role, Lon, Lat）；读 link.csv → `E_0`。
- 连通分量分析：找出现有环及其挂接的 ASG（环检测 / 双连通分量）。
- `C_new` = 度为 0 的 CSG。
- 构建成本查询函数 `cost(u,v)`（§1.1），road_distance 命中优先，GIS 兜底。

### S1 候选边生成（降复杂度）
- 全量点对 `O(|V|²)` 过大，做**空间稀疏化**：
  - 用 GIS 经纬度建 KD-Tree，每个孤点取地理最近的 `k`（建议 8~15）个邻居（邻居含 ASG、`C_old`、其它 `C_new`）。
  - 对候选邻居用 road_distance 重排（路网优先），剔除 `> D_max` 者。
- 输出候选边集 `E_cand`，规模约 `O(k·|C_new|)`。

### S2 现网盘点
- 每个现有环记录：`size`（当前节点数）、`slack = 10 − size`、锚定 ASG、环上各相邻边 `(a,b)`（作为可插入位）。
- 这些"可插入位 + slack"是 S3 的资源池。

### S3 构造式启发式（核心：容量受限最便宜插入）
对每个孤点 `x`，枚举两类落点，取**边际成本最小且可行**者：

1) **插入现有环**（优先，复用现网、成本低）：
   - 在所有 `slack>0` 的现有环上，对每条环边 `(a,b)` 计算插入代价
     `Δ = cost(a,x)+cost(x,b) − cost(a,b)`；
   - 取最小 `Δ` 的可行位（`size+1 ≤ 10` 且候选边在 `E_cand` 内）。

2) **并入/新建孤点环**（当现有环都满或太远）：
   - 把彼此邻近的孤点用容量约束聚类（如带容量上限 9 的地理聚类 / savings 算法）；
   - 每个簇构造**锚定最近 ASG 的环**：`ASG—(簇内最优哈密顿链)—ASG`（单 ASG 闭合）或挂到两个最近 ASG（双 ASG 环）；
   - 簇内链路由最近邻 + 2-opt 求近似最短哈密顿路。

> 处理顺序：按孤点的"接入难度"排序（离现网越远、可选落点越少者越先安置，避免被挤到高成本落点）。这是一种 regret-based 插入策略。

### S4 局部搜索改进（元启发式）
在满足约束前提下迭代降本：
- **2-opt**：单环内反转片段，消除交叉、缩短环长。
- **Or-opt / 节点迁移**：把某 CSG 从环 A 移到更近的环 B（B 有 slack），降总成本。
- **环间交换**：交换两环上各一节点。
- 每次移动后用 §1.3 环长 + §1.2 预言机做增量可行性检查；可包一层模拟退火/禁忌避免局部最优。

### S5 可行性校验（硬保障）
- 超级汇 `t` + 单位容量，逐 CSG 跑 `maxflow ≥ 2`。
- 任何不达标节点 → 定向修复：为其补一条到最近异向落点的边，或回滚最近一次破坏其双归的 splice。
- 同时复检：所有环 ≤ 10、节点度 ≤ 端口上限、无重复/自环边。

### S6（可选）MILP 精确模型（小规模/单簇求最优或评估启发式 gap）
- 决策变量 `y_e ∈ {0,1}`：是否选用候选边 `e`。
- 目标：`min Σ cost_e · y_e`。
- 约束（要点）：
  - 度约束：每个 CSG 在解中度 = 2（环上）；ASG 度 ≤ 端口上限。
  - 连通/双归：用单商品/多商品流约束，强制每个 CSG 向 ASG 集合送出 2 单位边不相交流。
  - 环长 ≤ 10：用流量标号 / MTZ 类约束限制从 ASG 出发的环跳数 ≤ 上限。
- 适合对每个聚类（≤9 CSG）单独求解，规模可控；整网用启发式。

## 4. 两个核心约束如何被保证

| 约束 | 保证机制 |
|---|---|
| 环节点 ≤ 10（口径①） | S2 维护 slack；S3 插入/建环时硬校验 `size ≤ 10`；S4 移动后增量校验 |
| 双向到达 ASG（口径②） | 解结构为"经过 ASG 的环"→ 天然 2 条边不相交路径；S5 用超汇 max-flow=2 全量兜底校验 |
| 成本最小 / 就近 | cost 以路网距离为主、GIS 兜底；S1 kNN 稀疏化保证只在近邻里选；S3 最便宜插入 + S4 局部搜索降本 |

## 5. 复杂度与规模
- S1 候选边：`O(|V| log|V|)` 建 KD-Tree + `O(k·|C_new|)` 候选。
- S3 构造：每个孤点 `O(候选落点数)`，总体近线性于候选边规模。
- S5 校验：每节点一次 max-flow（容量=2 时极快，2 次增广即停），`O(|C|·|E|)` 量级。
- 整体对接入层千级节点规模可在秒~分钟级跑完；MILP 仅用于小簇。
